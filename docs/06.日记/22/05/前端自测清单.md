# 前端自测清单

## 为什么
保证开发质量， 项目稳定，靠谱
- 都自测通过了，还要测试干嘛
>自测的目的是确保基本功能的用例正常，确保开发和产品对需求的理解一致，但仍有一些问题开发和产品都没有发现的，而测试就能够发现，毕竟测试是专业的，有些东西不能过于相信自己。

## 清单

- 根据需求写用例，把用例走一遍
- 把用例发给产品过一遍，对齐需求，避免遗漏
- 开发环境、测试环境、体验环境是否相同表现
- 代码CR,重要的问题提前发现


## 常用检查项
- 是否检查边界情况（如昵称过长或数值多大是否引起界面异常）；
- 浏览器兼容性检查（如IE,Firefox,Chrome或移动端手Q版本兼容性检查）；
- 接口异常处理（如接口异常是否提供重试或系统好友提示处理，减少用户投诉）；
- 是否涉及XSS、CSRF、Url含敏感字符串等安全问题（如涉及cgi需通过安全扫描）；
- 是否模拟低网速测试（如2G网络，可以用chrome或fiddler等工具模拟）；
- 是否有统计或日志上报（指UV,PV，需求里的统计和异常日志上报等等）；
- 前端资源是否可以压缩或合并（确保前端页面性能）；

- 全局组件，要多测试全局状态
